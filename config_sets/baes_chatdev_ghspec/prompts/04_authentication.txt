Implement comprehensive data validation and error handling.

Requirements:
- Add input validation for all POST and PATCH endpoints
- Email validation: must match standard email format (use regex or pydantic)
- Name validation: must be non-empty, max 100 characters
- Course code validation: must be alphanumeric, 3-10 characters
- Credits validation: must be positive integer between 1 and 12
- Department validation: must be non-empty, max 50 characters
- Grade validation: must be one of ['A', 'B', 'C', 'D', 'F'] or null
- Return appropriate HTTP status codes:
  - 400 Bad Request for validation errors
  - 404 Not Found for non-existent resources
  - 409 Conflict for duplicate email/course code
  - 422 Unprocessable Entity for invalid data types
- Return detailed error messages in JSON format with field-specific errors
- Add database constraint violations handling
